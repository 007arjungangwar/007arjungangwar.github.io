Practice Python like a coding platform

Students submit code, the backend runs test cases, and every result is stored for later use.

This page is the frontend prototype for a HackerRank-style flow. The actual evaluation happens in the Python backend added to this repo. For now it supports trusted local development, not public internet-safe code execution.

  • Try A Submission

How This Prototype Works

Student Submission

The student enters a name, picks a challenge, and writes Python code that implements the required function.

Test Case Evaluation

The backend runs the code against test cases with a timeout and returns pass or fail details for each case.

Submission Storage

Each result is saved into a JSONL file so you can later analyze attempts, accuracy, and student progress.

Future Data Export

You can later sync the saved data to Google Drive, Sheets, or a real database after the judging flow is stable.

Sample Challenges

Sum Two Numbers

Write a function named solve(a, b) that returns the sum of two integers.

Reverse A String

Write a function named solve(text) that returns the reversed string.

Before You Deploy Publicly

Do not expose this prototype directly on the internet yet. Running arbitrary Python from untrusted students requires containers, sandboxing, CPU and memory limits, filesystem isolation, and stronger auth than a local demo backend.

Backend Steps

1. Install Python dependencies from backend/requirements.txt.

2. Run python app.py inside the backend folder.

3. Submissions will be stored in backend/data/submissions.jsonl.
